    # swift oc混编

    # 先介绍下背景，公司各bu都拆成.a依赖。所以在混编swift的时候遇到一小点问题，今天记录下。

    # Library Search Paths加上 $(TOOLCHAIN_DIR)/usr/lib/swift/$(PLATFORM_NAME)，不然会遇到swift library link不到的情况。

    # Runpath Search Paths加了/usr/lib/swift,不然启动时会崩溃

    # 此外，我还尝试了新建一个默认语言为oc的.a作为bundle加入到主工程依赖，然后oc bundle进行调用。在加入swift代码的时候，会提示新建一个bridge文件，同时工程会隐式新建一个xxx-swift.h。 如果在一个工程内，那么直接import这个头文件没问题。但是我在其他的oc bundle直接引用，在编译时候就会报错，提示xxxx-swift.h找不到。

    # 找到方案是swift的bundle加入script在build的时候将xxx-swift.h copy到根目录

    target_dir=${BUILT_PRODUCTS_DIR}/include/${PRODUCT_MODULE_NAME}/ mkdir -p ${target_dir} cp ${DERIVED_SOURCES_DIR}/*-Swift.h ${target_dir}(xcode 11改为../*-Swift-5.0.h ${target_dir} )

    # 同时要将此bundle的define moudle设为yes
